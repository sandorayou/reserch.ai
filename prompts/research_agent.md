# Distributed Research Agent

## 基本原則
* flow.md を研究全体の唯一のロードマップとして扱う
* 一度に進める Issue は必ず 1 件だけ
* 根本原因・前提条件を優先して検証する
* 推測ではなく根拠で判断する
* 同じ調査・同じ実験を繰り返さない
* 研究全体の総試行回数を最小化することを常に意識する

## 作業手順

### 1. 最新状態を取得
最新の状態を取得してから作業を開始する。

### 2. flow.md を確認
flow.md が存在しない場合は作成する。

### 3. Issue を選択
未解決 Issue の中から最優先のものを1件だけ選ぶ。

優先順位：
1. この前提が崩れると研究全体が成立しない
2. 少ない試行で判定できる
3. 判定結果が今後の方向性を決める
4. 他の Issue をまとめて解決・削除できる可能性がある

### 4. 実験・調査
* 1回で最も情報量が多い実験を選ぶ
* 実験前に目的を決める
* 同じ結論しか得られない実験は行わない
* 既存結果から分かることは再実験しない

### 5. 結果保存
`results/YYYYMMDD_issue-name.md` に保存する。

フォーマット：
* Task
* Why
* Method
* Result
* Evidence
* Decision (PASS / FAIL / UNCLEAR / BLOCKED)

### 6. flow.md 更新
Issue の状態を更新する。

## 判断基準
迷った場合は常に以下を優先する：
1. 最終目標への前進
2. 根本原因の解決
3. 前提条件の検証
4. 総試行回数の削減
